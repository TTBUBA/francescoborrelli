⚡ Ottimizzazioni Unity per Giochi Mobili

Pubblicato il 5 Giugno 2026 | Lettura: 7 minuti

Introduzione

Le performance sono cruciali per i giochi mobili. In questo articolo scoprirai le tecniche più efficaci per ottimizzare il tuo gioco Unity e ridurre il consumo di risorse.

1. Object Pooling

Invece di istanziare e distruggere oggetti continuamente, riutilizzali:


public class BulletPool : MonoBehaviour
{
    public GameObject bulletPrefab;
    private Queue<GameObject> bulletPool = new Queue<GameObject>();
    public int poolSize = 50;

    void Start()
    {
        for (int i = 0; i < poolSize; i++)
        {
            GameObject bullet = Instantiate(bulletPrefab);
            bullet.SetActive(false);
            bulletPool.Enqueue(bullet);
        }
    }

    public GameObject GetBullet()
    {
        if (bulletPool.Count > 0)
        {
            GameObject bullet = bulletPool.Dequeue();
            bullet.SetActive(true);
            return bullet;
        }
        return Instantiate(bulletPrefab);
    }

    public void ReturnBullet(GameObject bullet)
    {
        bullet.SetActive(false);
        bulletPool.Enqueue(bullet);
    }
}

2. Disabilita Script non Necessari

Disabilita gli script quando non servono:


// Disabilita il componente quando il giocatore è lontano
if (Vector3.Distance(transform.position, player.position) > 100f)
{
    GetComponent<EnemyAI>().enabled = false;
}
else
{
    GetComponent<EnemyAI>().enabled = true;
}

3. Usa LOD (Level of Detail)

Riduci i dettagli grafici in base alla distanza dalla camera:


// LOD Group
LODGroup lodGroup = GetComponent<LODGroup>();
// Configura nel pannello Inspector:
// LOD 0 (0% - 50%): Modello ad alta qualità
// LOD 1 (50% - 80%): Modello media qualità
// LOD 2 (80% - 100%): Modello bassa qualità

4. Caching dei Componenti

Evita di cercare componenti ogni frame:


// CATTIVO - Lento
void Update()
{
    GetComponent<Rigidbody>().velocity = speed;
}

// BUONO - Veloce
private Rigidbody rb;

void Start()
{
    rb = GetComponent<Rigidbody>();
}

void Update()
{
    rb.velocity = speed;
}

5. Usa Physics.IgnoreLayerCollision

Riduci i calcoli di fisica ignorando collisioni non necessarie:


// Nel Start() del manager
Physics.IgnoreLayerCollision(LayerMask.NameToLayer("Player"), 
                             LayerMask.NameToLayer("Terrain"));

6. Gestione della Memoria

  • ✓ Evita foreach nei loop critici
  • ✓ Usa struct invece di class quando possibile
  • ✓ Riduci allocazioni di memoria nel frame update
  • ✓ Usa NativeArrays per dati numerici

Conclusione

Applica queste tecniche per migliorare drasticamente le performance del tuo gioco. Ricorda di profilare sempre con Unity Profiler! 📊
